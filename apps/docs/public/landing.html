Four Primitives

Simple building blocks. Compose them however you want. No complex graphs. No black boxes.

Harness

The execution system that gives agents time dimensionality. Knows what step you're on. Persists state across steps.

Agent

Autonomous decision-maker. Takes actions, returns results. Aware of step number, step history, constraints.

Step

A single execution point in time. The fundamental unit. Time-based, event-based, or task-completion based.

State

Persistent semantic memory. Survives step boundaries. Goal, progress, action log, knowledge graph.

Not LangChain

Django/Rails philosophy. Opinionated primitives, not infinite flexibility. Productive from day one.

What It's For

Out-of-loop agents that run autonomously for hours, days, weeks. You set up, leave, return to results.

Trading Bots

24/7 execution. Time-based steps. Complex state: portfolio, trades, strategy. Transparent audit log.

Coding Agents

Task-completion steps. State persists across sessions. Test-fix loops become step iterations. 200+ ticket progress tracking.

Research Agents

Information processing steps. Knowledge accumulation. Checkpoint when insights emerge. Persistent research state.

Monitoring Agents

Event-based steps. Step when conditions met. Alert history persists. Stateful monitoring patterns.

What It's Not

Clear boundaries. We do one thing well.

Not a Chatbot Framework

Use Vercel AI SDK for chat. We're for out-of-loop autonomous agents.

Not Replacing Anthropic SDK

We wrap it, extend it, add time awareness.

Not In-Loop

We don't do conversational, human-in-the-loop execution.

Not a Workflow Engine

No graphs, no DAGs. Just harness.step()—you control cadence.
