The conformance model defines how implementations must behave to be compatible with Open Harness.

## Conformance Levels

1. **Protocol conformance**: Must implement the kernel protocol (Hub, FlowRuntime, Agent, Channel)
2. **Flow conformance**: Must execute FlowSpec YAML correctly
3. **Test conformance**: Must pass all conformance tests

## Testing Tiers

- **Unit tests**: Test individual components
- **Replay tests**: Test flows using fixtures
- **Live tests**: Integration tests with real services
