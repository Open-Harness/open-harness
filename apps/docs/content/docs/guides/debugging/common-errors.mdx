# Common Errors Reference

Quick fixes for frequently encountered errors.

## Parse Errors

### Invalid YAML Syntax

```
Error: YAML parse error at line 5
```

**Cause**: YAML syntax error (indentation, colons, quotes).

**Fix**: Check indentation and special characters:

```yaml
# Bad
nodes:
- id: my-node
type: constant  # Wrong indent

# Good
nodes:
  - id: my-node
    type: constant
```

### Missing Required Field

```
Error: Missing required field: flow.name
```

**Cause**: Flow header missing required properties.

**Fix**: Add the required field:

```yaml
flow:
  name: my-flow  # Required
  version: "1.0"
```

## Validation Errors

### Node Type Not Found

```
Error: Unknown node type: my-custom-node
```

**Cause**: Node type not registered before execution.

**Fix**: Register the node type:

```typescript
const registry = new NodeRegistry();
registry.register(myCustomNode);
```

### Circular Dependency

```
Error: Circular dependency detected: A -> B -> A
```

**Cause**: Nodes depend on each other in a cycle.

**Fix**: Remove the cycle by restructuring edges:

```yaml
# Bad
edges:
  - from: A
    to: B
  - from: B
    to: A  # Creates cycle

# Good - break the cycle
edges:
  - from: A
    to: B
```

## Binding Errors

### Unable to Resolve Binding

```
Error: Unable to resolve binding: nodeX.output.value
```

**Cause**: Referenced node hasn't executed or doesn't exist.

**Fix**:
1. Check node ID spelling
2. Add missing edge to ensure execution order
3. Verify output structure

```yaml
edges:
  - from: nodeX
    to: nodeY  # Ensures nodeX runs first
```

### Type Mismatch

```
Error: Expected string, got object at path: input.text
```

**Cause**: Binding returns wrong type for schema.

**Fix**: Adjust binding path or schema:

```yaml
# If nodeX.output.data is { text: "hello" }
input:
  text: "{{nodeX.output.data.text}}"  # Access nested value
```

## Runtime Errors

### Node Execution Failed

```
Error: Node 'my-node' failed: Connection refused
```

**Cause**: Node threw an error during execution.

**Fix**: Handle errors in node implementation:

```typescript
async run(ctx, input) {
  try {
    return await riskyOperation();
  } catch (error) {
    ctx.hub.emit({ type: "narrative", text: `Error: ${error.message}` });
    throw error;
  }
}
```

### Timeout

```
Error: Execution timeout after 30000ms
```

**Cause**: Flow or node took too long.

**Fix**: Increase timeout or optimize:

```typescript
await executeFlow(flow, {
  input,
  registry,
  hub,
  timeout: 60000  // 60 seconds
});
```

## Schema Errors

### Input Validation Failed

```
Error: Input validation failed: text must be at least 1 character
```

**Cause**: Input doesn't match schema requirements.

**Fix**: Ensure input matches schema:

```typescript
const InputSchema = z.object({
  text: z.string().min(1)
});

// Provide valid input
await executeFlow(flow, {
  input: { text: "valid" }  // Not empty string
});
```

### Output Validation Failed

```
Error: Output validation failed at node 'transform'
```

**Cause**: Node returned data that doesn't match output schema.

**Fix**: Ensure node returns correct shape:

```typescript
async run(ctx, input) {
  return {
    result: "value",
    count: 42  // Include all required fields
  };
}
```
