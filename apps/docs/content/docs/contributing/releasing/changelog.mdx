# Changelog

The changelog documents all notable changes. This page covers the changelog process.

## Changelog Format

We follow [Keep a Changelog](https://keepachangelog.com/):

```markdown
# Changelog

All notable changes to this project will be documented in this file.

## [Unreleased]

### Added
- New feature description

### Changed
- Change description

### Deprecated
- Deprecation notice

### Removed
- Removal notice

### Fixed
- Bug fix description

### Security
- Security fix description

## [1.2.0] - 2024-01-15

### Added
- WebSocket channel support (#123)
- Retry policies for flow nodes (#124)

### Fixed
- Binding resolution for nested arrays (#125)
```

## Categories

### Added

New features:

```markdown
### Added
- `createOpenAIAgent` provider for OpenAI integration
- `retry` policy option for flow nodes
- WebSocket channel in `@open-harness/websocket-channel`
```

### Changed

Changes to existing functionality:

```markdown
### Changed
- `Hub.emit()` now returns void instead of boolean
- Default max tokens increased from 1024 to 4096
- Improved error messages for binding resolution
```

### Deprecated

Soon-to-be removed features:

```markdown
### Deprecated
- `Hub.notify()` is deprecated, use `Hub.emit()` instead
- `flow.timeout` option, use `policy.timeout` instead
```

### Removed

Removed features (breaking):

```markdown
### Removed
- Removed `Hub.broadcast()` method
- Removed support for legacy binding syntax
```

### Fixed

Bug fixes:

```markdown
### Fixed
- Fixed race condition in hub event emission (#45)
- Resolved memory leak in long-running flows (#46)
- Corrected binding resolution for optional paths
```

### Security

Security fixes:

```markdown
### Security
- Updated dependencies to address CVE-2024-1234
- Added input sanitization for flow definitions
```

## Adding Entries

### When to Add

Add changelog entries when:
- Adding new features
- Fixing bugs
- Making breaking changes
- Deprecating features

Don't add entries for:
- Internal refactoring (no visible change)
- Documentation updates
- Test additions

### How to Add

1. Open CHANGELOG.md
2. Add entry under [Unreleased]
3. Use correct category
4. Reference issue/PR if applicable

```markdown
## [Unreleased]

### Fixed
- Binding resolution for nested arrays (#125)
```

### Entry Format

```markdown
- Brief description of change (#issue)
```

Keep entries:
- Concise (one line)
- User-focused (what changed, not how)
- Referenced (#issue or #PR)

## Release Process

### Prepare Release

1. Create release branch:
   ```bash
   git checkout -b release/1.2.0
   ```

2. Update CHANGELOG.md:
   ```markdown
   ## [1.2.0] - 2024-01-15

   (move Unreleased content here)

   ## [Unreleased]

   (empty for now)
   ```

3. Update version in package.json files

4. Commit:
   ```bash
   git commit -am "chore: prepare v1.2.0 release"
   ```

### Create Release

1. Create tag:
   ```bash
   git tag v1.2.0
   ```

2. Push:
   ```bash
   git push origin v1.2.0
   ```

3. CI handles npm publish

### After Release

1. Update links at bottom of CHANGELOG.md:
   ```markdown
   [Unreleased]: https://github.com/.../compare/v1.2.0...HEAD
   [1.2.0]: https://github.com/.../compare/v1.1.0...v1.2.0
   ```

2. Merge release branch to main

## Example Changelog

```markdown
# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/).

## [Unreleased]

### Added
- OpenAI provider support

## [1.2.0] - 2024-01-15

### Added
- WebSocket channel package
- Retry policies for flow nodes (#123)
- `evaluateWhen` now supports `or` operator

### Changed
- Improved error messages for binding resolution
- Hub now uses AsyncLocalStorage for context

### Fixed
- Race condition in concurrent node execution (#125)
- Memory leak in long-running harness instances

## [1.1.0] - 2024-01-01

### Added
- Flow compilation with `compileFlow`
- Node lifecycle events

### Deprecated
- `Hub.notify()` is deprecated, use `Hub.emit()` instead

## [1.0.0] - 2023-12-15

### Added
- Initial stable release
- Hub event bus
- Harness runtime
- Flow execution
- Anthropic provider

[Unreleased]: https://github.com/org/repo/compare/v1.2.0...HEAD
[1.2.0]: https://github.com/org/repo/compare/v1.1.0...v1.2.0
[1.1.0]: https://github.com/org/repo/compare/v1.0.0...v1.1.0
[1.0.0]: https://github.com/org/repo/releases/tag/v1.0.0
```

## Best Practices

### Be User-Focused

```markdown
# Good
- Fixed crash when processing empty arrays

# Not as good
- Added null check in arrayProcessor.ts line 45
```

### Be Specific

```markdown
# Good
- `createHub` now accepts optional `sessionId` parameter

# Not as good
- Improved hub creation
```

### Group Related Changes

```markdown
### Added
- WebSocket channel support
  - Real-time event streaming
  - Bidirectional messaging
  - Connection management
```
