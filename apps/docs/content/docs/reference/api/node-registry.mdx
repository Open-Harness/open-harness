# Node Registry

The `NodeRegistry` stores node type definitions used by the runtime.

## Creating a Registry

```typescript
import { DefaultNodeRegistry } from "@open-harness/core";

const registry = new DefaultNodeRegistry();
```

## Methods

### `register(def)`

```typescript
registry.register(customNode);
```

### `get(type)`

```typescript
const def = registry.get("echo");
```

### `has(type)`

```typescript
if (registry.has("claude.agent")) {
  // ...
}
```

## Built-in Nodes

The SDK ships with these node definitions:

| Type | Export | Description |
|------|--------|-------------|
| `claude.agent` | `claudeNode` | Claude agent node |
| `constant` | `constantNode` | Emits a constant value |
| `echo` | `echoNode` | Echoes text input |

## Registering Custom Nodes

```typescript
import type { NodeTypeDefinition } from "@open-harness/core";

const customNode: NodeTypeDefinition<{ input: string }, { output: string }> = {
  type: "custom.processor",
  run: async (_ctx, input) => ({ output: input.input }),
};

registry.register(customNode);
```

## Using with Runtime

```typescript
const runtime = createRuntime({ flow, registry });
```
