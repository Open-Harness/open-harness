# Node Registry

The NodeRegistry manages available node types for flows.

## Creating a Registry

```typescript
import { DefaultNodeRegistry } from "@open-harness/sdk";

const registry = new DefaultNodeRegistry();
```

## Methods

### `register(node)`

Register a node type:

```typescript
registry.register(myAgent);
registry.register(httpGetNode);
```

### `get(name)`

Get a registered node:

```typescript
const node = registry.get("claude.agent");
```

### `has(name)`

Check if a node exists:

```typescript
if (registry.has("my-agent")) {
  // ...
}
```

### `list()`

List all registered nodes:

```typescript
const nodes = registry.list();
// ["claude.agent", "http.get", "my-agent"]
```

## Built-in Nodes

The kernel includes these nodes:

| Name | Description |
|------|-------------|
| `claude.agent` | Claude AI agent |
| `http.get` | HTTP GET request |
| `http.post` | HTTP POST request |
| `map` | Parallel array processing |
| `delay` | Wait for duration |

## Registering Custom Nodes

```typescript
import { defineAgent } from "@open-harness/sdk";

const customAgent = defineAgent({
  name: "custom.processor",
  async execute(input, ctx) {
    return { result: process(input) };
  },
});

registry.register(customAgent);
```

## Using with Hub

```typescript
const hub = createHub("session", {
  registry,
  store,
});

// Flows can now use registered nodes
await hub.startFlow("my-flow", input);
```
