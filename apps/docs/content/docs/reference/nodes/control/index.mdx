# Control Nodes

Control nodes manage flow execution: branching, looping, merging, waiting, and human-in-the-loop interactions.

## Available Control Nodes

| Node | Status | Description |
|------|--------|-------------|
| [`control.foreach`](/docs/reference/nodes/control/foreach) | Implemented | Iterate over arrays |
| [`control.if`](/docs/reference/nodes/control/if) | Planned | Binary branching |
| [`control.switch`](/docs/reference/nodes/control/switch) | Planned | Multi-way branching |
| [`control.merge`](/docs/reference/nodes/control/merge) | Planned | Merge parallel branches |
| [`control.loop`](/docs/reference/nodes/control/loop) | Planned | While-style loops |
| [`control.gate`](/docs/reference/nodes/control/gate) | Planned | Human-in-the-loop prompts |
| [`control.subflow`](/docs/reference/nodes/control/subflow) | Planned | Execute nested flows |
| [`control.wait`](/docs/reference/nodes/control/wait) | Planned | Delay or event wait |
| [`control.fail`](/docs/reference/nodes/control/fail) | Planned | Fail the run explicitly |
| [`control.noop`](/docs/reference/nodes/control/noop) | Planned | No-op passthrough |

## Usage Pattern

```yaml
nodes:
  - id: process-items
    type: control.foreach
    input:
      items: "{{flow.input.data}}"
      nodeId: process-single
```
