# GitHub Channel

The GitHub channel renders flow execution to a GitHub Issue, updating in real-time with formatted markdown.

## Installation

```bash
bun add @open-harness/github-channel
```

## Usage

```typescript
import { GitHubChannel } from "@open-harness/github-channel";

const runner = createFlowRunner(flow, registry);
runner.attach(GitHubChannel({
  owner: "my-org",
  repo: "my-repo",
  issue: 123,
}));

await runner.run();
```

## Features

- **Real-time updates**: Issue body updates as flow progresses
- **Markdown formatting**: Rich output with headers, code blocks, status indicators
- **Collapsible sections**: Phases collapse when complete
- **Progress tracking**: Visual progress indicators
- **Error reporting**: Failed tasks highlighted with details

## Configuration

```typescript
GitHubChannel({
  // Required
  owner: string;           // Repository owner
  repo: string;            // Repository name
  issue: number;           // Issue number to update

  // Optional
  token?: string;          // GitHub token (defaults to GITHUB_TOKEN env)
  updateInterval?: number; // Minimum ms between updates (default: 1000)
});
```

## Environment Variables

| Variable | Description |
|----------|-------------|
| `GITHUB_TOKEN` | GitHub personal access token with `issues:write` scope |

## Output Format

The channel renders a dashboard-style view:

```markdown
## Flow: research-pipeline

### Phase 1: Research ✓
<details>
<summary>3 tasks completed</summary>

- ✓ fetch-sources (2.3s)
- ✓ extract-data (1.8s)
- ✓ validate-schema (0.5s)

</details>

### Phase 2: Analysis ▶
- ▶ analyze-trends (running...)
- ○ generate-report (pending)
```
