# Contributing

**Status:** Outline Only  
**Purpose:** How to contribute to Open Harness

---

## Overview

How to contribute to the Open Harness codebase and ecosystem.

---

## Sections

### Development Setup

- Clone repository
- Install dependencies
- Run tests
- Build for distribution

### Code Style

- TypeScript best practices
- Neverthrow Result types (no exceptions)
- Function naming conventions
- Documentation standards (JSDoc)

### Testing

- Unit tests (per package)
- Integration tests (real SDK calls, no mocks)
- Recording fixtures (from live runs)
- Contract tests (store interfaces)

### Documentation Standards

- README files (package-level, module-level)
- YAML frontmatter (metadata with timestamps)
- JSDoc (inline documentation)
- Examples (working, tested)

### Pull Request Process

- Branch from dev
- Make changes
- Run quality gates (typecheck, lint, test)
- Submit PR to dev
- Code review process

Guide for contributing to core codebase and ecosystem.
